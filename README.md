# RAG-система для анализа документа «ПОПАТКУС»

Исследовательский проект по построению и сравнению стратегий чанкинга в RAG-системе на основе векторной базы данных ChromaDB с оценкой качества через RAGAS.

## Описание

Проект реализует полный пайплайн Retrieval-Augmented Generation (RAG) для работы с документом «ПОПАТКУС». В рамках исследования сравниваются **5 стратегий чанкинга**:

| Стратегия | Описание |
|-----------|----------|
| `recursive` | Рекурсивное разбиение текста с учетом структуры |
| `markdown_header` | Разбиение по заголовкам Markdown |
| `parent_document` | Иерархический чанкинг с родительскими документами |
| `semantic` | Семантическое разбиение на основе эмбеддингов |
| `legal_document` | Специализированная стратегия для юридических/структурированных документов |

Оценка качества производится с помощью фреймворка **RAGAS** по следующим метрикам:

- Context Relevancy
- Answer Relevancy
- Faithfulness

## Установка и запуск

### Требования

- Python 3.10+
- UV (рекомендуется) или pip

### Установка зависимостей

```bash
# Установка UV (если не установлен)
pip install uv

# Создание виртуального окружения
uv venv

# Активация окружения

# Linux / macOS
source .venv/bin/activate

# Windows
.venv\Scripts\activate

# Установка зависимостей
uv pip install -r requirements.txt

## Этапы работы

### 1. Препроцессинг документа

```bash
uv run main.py
```

Выполняет:

- загрузку документа «ПОПАТКУС»;
- очистку текста;
- предварительную обработку и подготовку данных.

---

### 2. Создание коллекций ChromaDB

```bash
uv run create_collections.py
```

Создаёт отдельные коллекции в ChromaDB для каждой из 5 стратегий чанкинга.

Коллекции именуются в соответствии с названиями стратегий.

---

### 3. Оценка стратегий (RAGAS)

```bash
uv run eval_strat.py
```

Запускает сравнительную оценку всех стратегий чанкинга с использованием RAGAS.

Результаты сохраняются в файл:

```text
evaluation_results.csv
```

---

### 4. Запуск Telegram-бота

```bash
uv run telegram_bot.py
```

Telegram-бот реализует полный RAG-пайплайн:

1. Принимает вопрос пользователя;
2. Выполняет поиск релевантных чанков в выбранной коллекции;
3. Генерирует ответ на основе найденного контекста;
4. Возвращает ответ с указанием использованной стратегии чанкинга.
